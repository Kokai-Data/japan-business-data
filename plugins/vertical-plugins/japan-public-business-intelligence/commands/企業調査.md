# /企業調査 (alias of /due-diligence)

日本企業の 1 ページ デューデリ brief を生成。法人番号または会社名で gBizINFO + 国税庁 法人番号 + EDINET (上場時) を citation 付きで引用、企業 profile / 財務 / 主要株主 / 異動履歴 / 上場開示 を整理。4 階層 authority strip + 士業 boundary disclaimer 付き。

使い方:

- `/企業調査 <13-digit 法人番号>` — direct (最速)
- `/企業調査 <会社名>` — 候補 search → user confirm

英語 alias: `/due-diligence` (同一機能、後方互換維持)

詳細 workflow は [due-diligence-jp agent](../../../agent-plugins/due-diligence-jp/agents/due-diligence-jp-agent.md) 参照。

## Boundary

Output は signal / 確認材料 / context — M&A 判断 / 投資判断 / 適格性判断 は別途専門家 (公認会計士 / 税理士 / 弁護士 / 中小企業診断士) 確認前提。Kokai は判定者ではなく signal 提示者。
